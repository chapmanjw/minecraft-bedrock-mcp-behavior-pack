# Contributing

Thanks for your interest in improving the Bedrock Bridge behavior pack.

## Development setup

```sh
npm install
```

## Checks

Every change must pass the same gates CI runs:

```sh
npm run typecheck    # strict TypeScript
npm run lint         # ESLint, type-checked rules
npm run format:check # Prettier
npm test             # vitest
npm run build        # bundles src/ into scripts/main.js
```

`npm run format` applies formatting, and `npm run test:watch` runs the tests in watch mode.

## The committed bundle

`scripts/main.js` is generated by `npm run build` and **committed** — releases ship it and CI
fails if it drifts from `src/`. Always run `npm run build` and commit the result alongside any
source change.

## Conventions

- TypeScript, strict mode, ESM. No `any` — prefer `unknown` and narrow.
- Small, focused modules; one responsibility per file. Domain-driven names — no `utils` or
  `helpers` grab-bags.
- Handlers are pure with respect to their payload. Every side effect goes through a named service
  on the `HandlerContext` — the job scheduler, the subscription manager, the event publisher.
  Handlers never touch `system`, the transport, or the network directly.
- The Script API is reached only through `JobScheduler`: `run` for a deferred mutation, `runJob`
  for a generator that yields across ticks. A handler that sweeps many blocks must yield.
- Tests exercise behavior through the mock Script API and in-memory fakes; they do not test
  private internals.

## The bridge protocol

`src/protocol/` is the wire contract with the MCP server and mirrors the server repository's
`protocol/` module file-for-file. Change it deliberately, in lockstep with the server, and bump
`PROTOCOL_VERSION` when the contract changes.

## Script API declarations

`types/` vendors the slice of the `@minecraft/server*` API the pack uses. When a handler needs a
new API, extend the relevant declaration to match the official Script API documentation.

## Pull requests

Keep pull requests focused on a single change. Describe what changed and how you verified it.
